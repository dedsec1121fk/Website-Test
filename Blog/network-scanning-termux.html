Network Scanning with Termux: A Beginner's Guide

Master the fundamentals of ethical network scanning using Termux. Learn to discover devices, identify open ports, and assess network security using professional tools from your Android device.

Termux & Hacking 14 min read 2026-01-13

Important Legal Disclaimer

Before we begin, it is absolutely critical to understand the legal implications of network scanning. Scanning networks without explicit authorization is illegal in most jurisdictions and can result in serious criminal charges, including violations of the Computer Fraud and Abuse Act (CFAA) in the United States or similar laws in other countries. Only scan networks that you own, networks you have written permission to test, or designated practice environments like your home lab.

This article is intended for educational purposes, helping aspiring security professionals understand network reconnaissance techniques. Always practice ethically and legally.

Understanding Network Scanning

Network scanning is the process of identifying active hosts, open ports, running services, and potential vulnerabilities within a computer network. For security professionals, it's an essential skill used during the reconnaissance phase of penetration testing. Understanding what's on your network is the first step toward securing it.

Types of Network Scans

There are several types of network scans, each serving different purposes. Host discovery scans identify which devices are active on a network. Port scans determine which network ports are open on a target device. Service scans identify what software is running on open ports. Vulnerability scans look for known security weaknesses in discovered services.

Setting Up Your Scanning Environment

Before starting any scans, we need to install the necessary tools in Termux. The primary tool we'll use is Nmap (Network Mapper), the industry-standard network scanning utility used by security professionals worldwide.

Installing Essential Tools

pkg update && pkg upgrade
pkg install nmap
pkg install net-tools
pkg install iproute2
pkg install whois
pkg install dnsutils

Finding Your Network Information

Before scanning, you need to understand your own network configuration. Use these commands to discover your IP address and network range:

ifconfig
# or
ip addr show

# Find your gateway
ip route | grep default

A typical home network might show your IP as 192.168.1.105 with a subnet mask of 255.255.255.0, meaning your network range is 192.168.1.0/24 (all addresses from 192.168.1.1 to 192.168.1.254).

Host Discovery: Finding Devices on Your Network

The first step in network reconnaissance is discovering what devices are connected to the network. Nmap offers several techniques for host discovery, each with different stealth levels and effectiveness.

Basic Ping Sweep

The simplest form of host discovery sends ICMP echo requests (pings) to each address in a range:

nmap -sn 192.168.1.0/24

The -sn flag tells Nmap to perform a ping scan only, without port scanning. This is fast and relatively non-intrusive. The output will show all responding hosts with their IP addresses and, when possible, their MAC addresses and vendor information.

ARP Scan for Local Networks

For local networks, ARP (Address Resolution Protocol) scanning is more reliable because firewalls cannot block ARP requests within the same subnet:

nmap -PR 192.168.1.0/24

Understanding the Output

A typical host discovery output looks like this:

Nmap scan report for 192.168.1.1
Host is up (0.0034s latency).
MAC Address: AA:BB:CC:DD:EE:FF (Router Manufacturer)

Nmap scan report for 192.168.1.105
Host is up (0.00012s latency).

Nmap scan report for 192.168.1.110
Host is up (0.025s latency).
MAC Address: 11:22:33:44:55:66 (Samsung Electronics)

Port Scanning Fundamentals

Once you've identified active hosts, the next step is determining what services they're running by scanning their ports. Network ports are numbered endpoints (0-65535) that services use to communicate. Well-known ports include 22 for SSH, 80 for HTTP, 443 for HTTPS, and 3306 for MySQL.

TCP Connect Scan

The most basic and reliable scan type completes the full TCP three-way handshake:

nmap -sT 192.168.1.1

This scan is reliable but also the most detectable because it creates full connections that are logged by the target system.

SYN Stealth Scan

The SYN scan is faster and stealthier, sending only the initial SYN packet and analyzing the response without completing the connection:

nmap -sS 192.168.1.1

Note: SYN scans require root privileges on most systems. In Termux without root, you may need to use TCP connect scans instead.

Scanning Specific Ports

# Scan specific ports
nmap -p 22,80,443 192.168.1.1

# Scan a range of ports
nmap -p 1-1000 192.168.1.1

# Scan all 65535 ports
nmap -p- 192.168.1.1

# Scan common ports (fast)
nmap -F 192.168.1.1

Service and Version Detection

Knowing a port is open is useful, but knowing what service is running and its version is invaluable for security assessment. Outdated software versions often have known vulnerabilities.

Version Detection Scan

nmap -sV 192.168.1.1

This command probes open ports to determine what service is running and attempts to identify the specific version. Output might show:

PORT     STATE SERVICE VERSION
22/tcp   open  ssh     OpenSSH 8.4 (protocol 2.0)
80/tcp   open  http    Apache httpd 2.4.51
443/tcp  open  ssl     OpenSSL 1.1.1
3306/tcp open  mysql   MySQL 5.7.36

Operating System Detection

nmap -O 192.168.1.1

OS detection analyzes TCP/IP stack characteristics to guess the target's operating system. This can reveal whether a device is running Windows, Linux, macOS, or even specific versions.

Comprehensive Scan

nmap -A 192.168.1.1

The -A flag enables aggressive scanning, combining OS detection, version detection, script scanning, and traceroute. This provides maximum information but is slower and more detectable.

Nmap Scripting Engine (NSE)

Nmap's Scripting Engine extends its capabilities with hundreds of scripts for specific tasks like vulnerability detection, exploitation, and information gathering.

Using NSE Scripts

# Run default safe scripts
nmap -sC 192.168.1.1

# Run specific script
nmap --script=http-headers 192.168.1.1

# Run vulnerability scripts
nmap --script=vuln 192.168.1.1

# List available scripts
ls /data/data/com.termux/files/usr/share/nmap/scripts/

Useful Scripts for Beginners

Some particularly useful scripts include http-enum for discovering web directories, ssh-auth-methods for identifying SSH authentication types, and ssl-enum-ciphers for analyzing SSL/TLS configurations:

nmap --script=http-enum 192.168.1.1
nmap --script=ssh-auth-methods 192.168.1.1
nmap --script=ssl-enum-ciphers -p 443 192.168.1.1

Saving and Analyzing Results

Proper documentation is crucial for security assessments. Nmap supports multiple output formats for different purposes.

Output Formats

# Normal output (human-readable)
nmap -oN scan_results.txt 192.168.1.0/24

# XML output (for tools and scripts)
nmap -oX scan_results.xml 192.168.1.0/24

# Grepable output (for parsing)
nmap -oG scan_results.gnmap 192.168.1.0/24

# All formats at once
nmap -oA scan_results 192.168.1.0/24

Practical Exercises

Practice these exercises on your own network to build your skills:

Exercise 1: Network Inventory

Create a complete inventory of your home network:

nmap -sn 192.168.1.0/24 -oN network_inventory.txt

Exercise 2: Router Analysis

Scan your router to understand its exposed services:

nmap -sV -p 1-1000 192.168.1.1

Exercise 3: Web Server Detection

Find all web servers on your network:

nmap -p 80,443,8080 --open 192.168.1.0/24

Security Considerations and Best Practices

When performing network scans, always follow these guidelines to stay ethical and legal:

First, always get written permission before scanning any network you don't own. Second, document everything—keep detailed logs of what you scanned and when. Third, start with less intrusive scans and only escalate if necessary. Fourth, be aware that aggressive scanning can disrupt services or trigger security alerts. Fifth, never attempt to exploit vulnerabilities you discover without explicit authorization.

Conclusion

Network scanning is a fundamental skill for anyone interested in cybersecurity. With Termux and Nmap, you have professional-grade tools in your pocket to learn and practice these techniques. Remember that with great power comes great responsibility—always use these skills ethically and legally.
