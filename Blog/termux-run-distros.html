Run Linux Distros in Termux Using Only Your Phone

Use proot containers to run Ubuntu/Debian/Alpine safely without root, plus practical performance tips.

← Back to Blog Install DedSec Project

1) What you’re really running (and what you’re not)

On Android you typically run a userspace container (proot). It feels like a distro, but it’s not a full VM: no systemd by default, no real kernel change, and it’s safer than rooting.

2) Install proot-distro

pkg update -y && pkg upgrade -y
pkg install -y proot-distro

proot-distro manages downloads, installs, and launches. It’s the simplest method for Ubuntu/Debian/Alpine in Termux.

3) Install and start a distro

proot-distro list
proot-distro install debian
proot-distro login debian

Inside the distro, use apt (or apk on Alpine). Keep installs minimal for speed—phones have limited RAM and storage.

4) When you need a GUI

For GUI apps you usually use Termux:X11 or VNC. Start with a lightweight desktop (XFCE/LXQt). Expect slower performance than a PC—optimize and keep sessions small.

  • Prefer CLI tools when possible
  • Close heavy apps to avoid crashes

5) Performance and stability tips

Use one distro per purpose. Don’t install everything “just in case”. Keep your phone cool, avoid background apps, and do updates when on Wi‑Fi.

6) Security mindset (important)

Treat distro containers like normal Linux: use strong passwords for services, don’t expose SSH to the internet from your phone, and avoid running random scripts as root inside the container.
