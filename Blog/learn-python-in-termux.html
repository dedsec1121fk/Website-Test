How to Start Learning Python Using Termux

A simple setup + practice plan so you actually make progress (even with little time).

Why you can trust this guide

Everything here is tested with real Termux workflows: it focuses on the commands you actually run, the errors you actually see, and the fastest safe fixes. Copy, paste, and verify with the checks included.

← Back to Blog Install DedSec Project

1) Install Python and create a workspace

pkg update -y && pkg upgrade -y
pkg install -y python git
mkdir -p ~/Projects/python
cd ~/Projects/python

Keep all Python projects in one folder so you can back them up easily and avoid losing files across random directories.

2) Your first script (and how to run it)

cat > hello.py << 'PY'
name = input("Name: ")
print("Hello,", name)
PY

python hello.py

If you can create a file, accept input, and print output, you already have the core loop of most small tools.

3) The 5 topics that unlock most scripts

Focus on: variables + types, if/else, loops, functions, and reading/writing files. With these, you can build scrapers, note apps, and automation.

  • Practice daily: 1 small program
  • Use problems, not theory-only reading

4) Debug like a pro (on a phone)

When a script breaks, print intermediate values, reduce the input, and reproduce the error reliably. Don’t guess—measure.

python -m pip install -U pip
python -X faulthandler your_script.py

5) A realistic weekly plan

Week 1: basics + files. Week 2: functions + modules. Week 3: requests + JSON. Week 4: one mini-project you can publish (even a small tool).

pkg update -y
pkg install -y python
python --version

python -m pip install requests

cat > check_site.py <<'PY'
import requests

url = input("URL (https://...): ").strip()
try:
    r = requests.get(url, timeout=10)
    print("Status:", r.status_code)
    print("Server:", r.headers.get("server", "?"))
except Exception as e:
    print("Error:", e)
PY

python check_site.py
