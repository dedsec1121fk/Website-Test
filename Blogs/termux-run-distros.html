Run Distros & OSes in Termux Using Only Your Phone

Use proot-distro to run Debian/Ubuntu/Alpine safely without root.

← Back to Blog Install DedSec Project

Why proot-distro (and what it is)

proot-distro lets you run Linux userlands (Debian/Ubuntu/Alpine) inside Termux without root. It’s perfect for dev tools that aren’t packaged for Termux.

It’s not a full VM: it’s a compatibility layer. Expect slower performance than native Termux.

Install and list available distros

pkg update -y && pkg upgrade -y
pkg install -y proot-distro

See what you can install:

proot-distro list

Install Ubuntu or Debian and login

Example with Ubuntu:

proot-distro install ubuntu
proot-distro login ubuntu

Example with Debian:

proot-distro install debian
proot-distro login debian

Inside the distro, use apt as usual:

apt update && apt upgrade -y

Share folders between Termux and the distro

You can access Termux home from inside the distro at: 

/data/data/com.termux/files/home

Pro tip: keep projects in Termux (~/Projects) and edit/run inside the distro if needed.

Performance tips (low-end phones)

  • Prefer Alpine for speed & size.
  • Close heavy apps in the background.
  • Use CLI editors (nano/vim) instead of big IDEs.

Install Alpine example:

proot-distro install alpine
proot-distro login alpine

What you can (and can’t) do inside proot

  • ✅ Install packages with apt/apk, run servers, compile code, use Python/Node/Go
  • ✅ Run CLI apps and most developer tools without root
  • ❌ No real kernel modules, no real systemd, limited low-level networking

Think of proot like a safe “Linux user-space sandbox” on your phone.

Make it feel like a real workstation

Inside the distro, install a few comfort tools:

apt update && apt install -y git curl wget nano vim build-essential

Want a nicer prompt? Add this (Debian/Ubuntu):

echo 'PS1="\u@\h:\w$ "' >> ~/.bashrc
source ~/.bashrc

Bonus: run a web server from your phone

This is a fun way to learn. Start a simple server and open it from your browser.

cd ~
python3 -m http.server 8000

Then visit: http://127.0.0.1:8000
